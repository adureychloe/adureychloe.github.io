leetcode 思路

Posted on 2023-09-08，8 min read

数组

  1. 在一个数组中找一个位置，想到先将数组排序后用二分
  2. 对一个数组操作后前后长度改变，想到用快慢指针，快指针一直移动，慢指针指向更新数组下标，满足条件才动
  3. 有序数组的平方后排序，用双指针分别指向开头和结尾，谁小在结果数组放谁。
  4. 求数组的连续子序列，一般滑动窗口。用一个for循环，索引是窗口的终止位置，当窗口内元素不满足要求了，就向前移动起始位置。
  5. 螺旋矩阵，要按照顺时针走四条边，每条都左闭右开。全部圈的循环次数看n//2，中间的位置也是n//2，从（0，0）开始第一圈，第二圈开始变成（1，1）
  6. 一个数组的连续子数组个数，比如[1,2,3]的子数组，当i指向-1，j遍历时，子数组个数就是j-(-1)个，比如0-(-1)是1，也就是[1]，1-(-1)是2，就是[1,2][2]，也就是只要一直加就能得到全部的子数组个数。（在某一区间内，第一个i指向左区间左边，第二个j遍历区间，一直加）

链表

  1. python链表节点：
    class ListNode:
    def __init__(self, val, next=None):
        self.val = val
        self.next = next
  2. 更改链表，使用dummy_head。
  3. 反转链表，用迭代或递归，用双指针，一个指向前一位置，一个指向当前位置。
  4. 反转链表区间，用三个指针，分别指向前、中、后位置，prev一直固定，指向不断变化的下一个位置，一直是cur向后移动（其实是cur指向的在动，cur指的数不变），next是cur后一个，每次循环将cur的下一个移动到prev下一个。
  5. 删除倒数第n个节点，用快慢指针，首先快指针走n步，然后快慢指针一起走，直到快指针指向空，慢指针就指向n（注意倒数第n个就是正数第 L-n 个）。
  6. 链表相交，先将长的对齐到短的，然后一起向前，遇到相同节点就返回。
  7. 环形链表，用快慢指针，环的起点就是快慢指针相遇之后，慢指针重新指向开头，快慢指针同速前进，下一个相遇点。
  8. 删除数组元素用双指针。

哈希表

  1. 哈希碰撞：

    • 拉链法：存储在链表中
    • 线性探测法：向下找空位

  2. 遇到需要判断一个元素是否出现过马上想到哈希

  3. 求交集用 set和&，或者字典哈希，或者数组

  4. 求快乐数，因为和可能会重复出现，只要一出现重复就可以判断不是快乐数，所以使用哈希。

  5. 两数之和，用字典，如果target-当前值的结果在字典中，返回，否则记录当前值。

  6. 四数相加，求四个数组各取一个和为0的可能次数。思路：

    • 首先一个哈希，key放前两个数组元素之和a+b，value放出现次数
    • 遍历后两个数组，找 0-(c+d) 在哈希中的出现

  7. 如果数据量大时间过不掉，就不用字典而用数组（前提是数组长度已知）

  8. 三数之和，求数组中a+b+c=0的个数，可以排序后用哈希确定 0-（a+b）在不在哈希中，注意如果第一个数大于0就绝对没有。

字符串

  1. 反转字符串用切片或reversed，不能用的话用双指针。
  2. 反转每2k的前k个，每次前进2k，用一个p记录2k，一个p2记录k，反转前k就行。
  3. 替换空格，用replace或join+split
  4. 翻转字符串里的单词，简单的就是split加倒序，如果不用辅助空间，先将全部字符反转，然后单词内部反转。
  5. 左旋字符串，直接用切片拼接
  6. 判断字符串是否可以由重复的子字符串组成，因为字符串如果由重复子字符串组成，可以从中间分成前后一样的两段s1+s2，如果在后面拼接上本身，中间就有s2+s1=s出现，注意要剔除第一和最后一个元素，保证s是出现在中间。即掐头去尾后用find。
  7. 很多数组填充类的问题，都可以先预先给数组扩容带填充后的大小，然后在从后向前进行操作

栈和队列

  1. 用栈模拟队列，需要两个栈，一个进栈，一个出栈。python中用列表模拟就行，pop方法就是最右边的元素。
  2. 用队列模拟栈，只需要一个队列，在模拟栈弹出元素的时候只要将队列头部的元素重新添加到队列尾部，只留一个，就可以模拟栈了。队列在python里是collections里的deque，使用popleft弹出，append添加
  3. 删除字符串中的所有相邻重复项，和括号匹配相似，用栈记录元素，如果出现就pop。
  4. 求滑动窗口中的最大值，使用单调队列。队列中的元素要排序，维护元素单调递减的队列就叫做单调队列，单调队列不是排序，只维护有可能成为最大值的元素，比如对于{2，3，5，1，4}，单调队列只维护{5，4}，出口元素就是最大元素。需要三个函数：pop、push和front（返回出口元素）。因此pop和push操作规则：如果窗口移除元素等于出口元素，就pop，否则不变；如果push的元素大于入口元素，就将入口元素pop，直到push元素小于入口元素。

二叉树

  1. 二叉树的递归遍历用栈，只要注意顺序不一样。迭代也是用栈，但是前中后不一样。前序遍历是中左右，每次先处理的是中间节点，那么先将根节点放入栈中，出栈，然后将右孩子加入栈，再加入左孩子（这样出栈才是左右）。中序遍历是左中右，先访问的是二叉树顶部的节点，然后一层一层向下访问，直到到达树左面的最底部，再开始处理节点（也就是在把节点的数值放进result数组中）。后续遍历是左右中，那么我们只需要调整一下先序遍历的代码顺序，就变成中右左的遍历顺序，然后在反转result数组，输出的结果顺序就是左右中了。

    递归三要素：
    确定递归函数的参数和返回值
    确定终止条件
    确定单层递归的逻辑

  2. 层序遍历用队列。

  3. 翻转二叉树用前序/后序/层序遍历。

  4. 判断对称二叉树，使用后序遍历，通过递归函数的返回值来判断两个子树的内侧节点和外侧节点是否相等。正是因为要遍历两棵树而且要比较内侧和外侧节点，所以准确的来说是一个树的遍历顺序是左右中，一个树的遍历顺序是右左中。

  5. 使用前序求的就是深度，使用后序求的是高度。

  • 二叉树节点的深度：指从根节点到该节点的最长简单路径边的条数或者节点数（取决于深度从0开始还是从1开始）
  • 二叉树节点的高度：指从该节点到叶子节点的最长简单路径边的条数或者节点数（取决于高度从0开始还是从1开始）

根节点的高度就是二叉树的最大深度，所以我们通过后序求的根节点高度来求的二叉树最大深度。

  1. 求最小深度，也可以用后序遍历求高度，不过要注意最小深度是到叶子节点：

    所以有条件，如果左子树为空，右子树不为空，说明最小深度是 1 + 右子树的深度。反之，右子树为空，左子树不为空，最小深度是 1 + 左子树的深度。 最后如果左右子树都不为空，返回左右子树深度最小值 + 1.
    求二叉树的最小深度和求二叉树的最大深度的差别主要在于处理左右孩子不为空的逻辑。

  2. 判断是不是满二叉树：完全二叉树中递归向左遍历的深度等于递归向右遍历的深度。
