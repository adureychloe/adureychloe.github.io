一、TTPDrill: Automatic and Accurate Extraction of Threat Actions from Unstructured Text of CTI Sources

摘要

缺少标准化语言和对网络威胁信息的自动分析技术，使得从CTI报告中分析非结构化文本非常耗时耗力。本文目标是设计一个自动化且考虑上下文的威胁情报提取器，从公共CTI来源中精确提取到攻击模式（TTP）。本文贡献有三个：

  • 创新性地提出一个威胁-动作本体。
  • 设计一个混合NLP和IR的文本挖掘方法，基于语义而不是句法关系抽取威胁动作。
  • 通过映射每个威胁动作到一个技术、战术和杀伤链阶段，构造出一个完整的攻击模式，并转换成威胁情报标准，如STIX2.1。
    TTPDrill在不同评估方式上实现了超过82%的精确率和召回率。

引言

TTPDrill 结构

首先爬取报告、清洗数据、丢弃无关文章（例如广告）；然后使用词形解析，从文章中提取候选威胁动作（subject、verb、object），这些动作之后利用BM25 TF-IDF赋予权重并筛选，找出与威胁本体最符合的威胁动作。报告中的IOC会对识别造成影响，所以要用正则替换：

威胁动作提取例子：

威胁动作本体

本体就是对一个特定领域的规范描述。我们的方法基于ATT&CK和CAPEC构造。

本体类型：

  • 威胁动作：中心种类
  • 战术：ATT&CK提供10种，CAPEC提供9种
  • 技术：战术下的具体攻击技术，一个技术可能被多个威胁动作执行

威胁动作提取和分析

TTPDrill期望一个报告对应一个恶意软件。

爬取和预处理

用SVM训练文档分类器，区分出无关文档。

清洗文本：

  • 去除HTML标签、图片、视频

文本分类：

SVM分类器特征：

  • 词语数量：TTP文章的词数量更多
  • 安全动作-词 密度：根据公共信息标准（ATT&CK、CAPEC、CWE、CVE）提取verb，然后为了标记这些verb，计算该密度，即这些verb在一篇文章中的出现比例
  • 安全目标-词 密度：计算安全noun的比例

正则匹配

威胁动作识别

分两步提取文章中的威胁动作：识别候选威胁动作、将候选动作映射到本体的威胁动作。

一个威胁动作包含一个SVO（subject, verb, object）语法结构，用斯坦福依赖解析器识别，源结点称为governor，目的结点称为dependent。

威胁动作映射和筛选

一个已知的本体entry表示成词袋，候选动作也表示成词袋，计算相似度，计算方法是BM25为权重函数的TF-IDF。注意要考虑同义词。

一对一的候选动作称为原始动作，多对一的动作称为复合动作，前者一对一计算相似度，后者将多个候选动作合并计算相似度，合并原则是：

  • 只考虑m窗口大小的句子提取的候选动作（m=3）
  • 只有相似度分数提升超过阈值才合并

构建TTP杀伤链

评估

两个阈值：

  • 候选与本体匹配的截断阈值
  • 合并选择阈值

Acing the IOC Game- Toward Automatic Discovery andAnalysis of Open-Source Cyber Threat Intelligence

摘要

IOC（如恶意软件指纹，僵尸网络ip）。提出iACE，自动提取IOC。

依据是文章中的IOC通常与通过语法关系与一系列上下文联系在一起，iACE自动定位IOC和上下文，通过图挖掘技术分析关系，最后生成OpenIOC项。

实验在45个博客的71000篇文章提取了900k的OpenIOC项，准确率95%，覆盖率90%。

引言

CTI经常用IOC形式，如病毒指纹、僵尸网络IP/域名，攻击文件MD5hash。

难点

NER难点在于一个领域的方法不适用于另一个领域，RE难点在于IOC和上下文之间的关系复杂。

整体结构

预处理文章：使用topic-term分类找出可能包含IOC的文章，ocr提取图片中的文字。

定位包含IOC token的句子，建立IOC token和上下文之间的联系，将句子转化为依赖图DG。

利用图挖掘技术分析token之间的关系，确认是否真的包含IOC。

HinCTI: A Cyber Threat Intelligence Modeling and Identification System Based on Heterogeneous Information Network

摘要

创建CTI很困难，因为CTI之间内在和外在的关系，还有CTI中包含的威胁基础设施节点的异构性。因为网络威胁基础设施节点标签有限，自动识别基础设施节点的威胁种类很困难。本文提出HinCTI来创建网络威胁情报和识别威胁类型。首先，设计了一个威胁情报元模式来描述基础设施节点间的语义关系。之后在HIN（异构信息网络）上创建威胁情报，能融合各种基建节点和它们之间的丰富关系。接下来，定义了MIIS指标，提出基于MIIS度量的异构图卷积网络（GCN）来识别基建节点威胁类型。

本文第一个在HIN上创建CTI来威胁识别并提出一个基于异构GCN的方法来识别威胁类型。

引言

本研究只考虑结构化的CTI数据。

意义

  • 很少有研究集中在CTI中包含的基建节点的有限威胁类型标签问题。
  • 很少研究关注从HIN的视角研究基建节点之间的深层语义关系。

贡献

  • 从计算的视角（基于元路径和元图物质计算）提出基于HIN的CTI创建方法
  • 基于MIIS度量的异构GCN方法来识别基建节点的威胁类型
  • 提出HinCTI系统，创建CTI和识别威胁类型

GapFinder: Finding Inconsistency of Security Information From Unstructured Text

研究领域现状和痛点

现状

One commonality among CTI methods is a central reliance on publicly available information.

痛点

Derive a quality metric that captures the consistency and accuracy of the web-mined information and used to drive a CTI service.

方法、创新点和重要细节

方法

  • unstructured text to structured data

    Construct malware graph based on entity and relation types.

  • Data refinement process

    • Normalize the format of the nodes.

    • Refine different words with a similar contextual meaning into a single representation.

  • Formulate semantic connections between malware aliases

    • linking same malware nodes that have different names.
    • finds an inconsistency between nodes of the same type in the connected graph

创新点

  • Coreference resolution algorithm: If malware name and type is separated in the article, the type will be labeled as "O". And if later sentence has the type, it won't be coreference to the malware. To address this, the algorithm is designed.

    • Label the malware's proper noun as "MW"
    • Look for a noun that indicates a malware type within three words back and forth around malware name found in first step, in this work there's four types(trojan, worm, botnet, ransomware), if there's none, default type is "malware"
    • If the malware type appears after the sentence containing a malware name, it's replaced with the malware name, and the label is modified from "O" to "MW"

  • IT companies can be wrongly annotated with "MW", so use K-means clustering to classify malware and non-malware.

  • obtain absolute time and formatted date nodes. e.g., "last September" should be transformed to "2014-09" since the writing date is 2015.

重要细节

实验结果和结论

  • Construct a groud truth dataset by manually annotating the entities and their relations, randomly select 1000 sentences.
  • Only focus malware, in the future will seek to expand the scope of relation-types.
  • The base date and set may be wrong, there are some ways to determine the truth from nodes, such as through a reputation-based weighting scheme.
  • Unstructured data has noise.

方法

提出基于关联增强的技战术分类模型RENet，使用BiRNN和CNN作为基础分类器，并通过使用自学习的技战术关联矩阵提高分类效果。

两个任务：

  • 多标签分类任务：两个独立分类器，一个分战术，一个分技术。权重和层不共享。
  • 使用关联增强修正技术：在分类器尾部使用关联增强方式将战术分类效果迁移至技术分类器。

创新点

使用关联增强将战术效果增强给技术效果。

mitre官网的14个战术和184个主要技术的6504条描述语句。

Towards the Detection of Inconsistencies in Public Security Vulnerability Reports

研究领域现状和痛点

现状

根据CVE ID，可以从NVD和其它论坛得到信息。

痛点

NVD上的信息可能不全或过时。

方法、创新点和重要细节

方法

提出VIEM来抽取漏洞软件名字和版本。面临三个问题：

  • 难以用字典或正则表达来抽取
  • 传统NLP难以解决
  • 除了有漏洞的软件，还有正常软件作为干扰

整个系统分为NER和RE。NER识别名字和版本，RE将句子中所有识别的名字和版本连接起来后找出最有可能的名字-版本对。数据集大量标注1种类型，其它类型少量，使用迁移学习来扩展标注。

VIEM结构：

NER模块首先用word embedding和character embedding，用Bi-GRU预测标签。

这里加入一个字典库匹配的方法来提高效果：

  • 字典库是CVE product list
  • 用字典库里的词进行查询，如果至少有一个词被NER模型标为名字，就将匹配的前后词都标为名字。如NER将Explorer标为名字，查询字典后将整个Internet Explorer标为名字。

RE模块首先将名字和版本编码index，然后将其一一对应，将其one-hot编码后再转换为两个向量，作为位置向量，和NER的word embedding一起送入两个注意力层得到最终的预测概率，即正确的名字-版本对。

对于不一致性，使用NVD项和CVE描述及漏洞报告进行对比。

首先匹配软件名字，匹配方法：

  • 如果两个名字的相同词数量大于或等于不同词数量，就说他们匹配（这一点可以改进，利用知识图谱的节点相似性）

对匹配的名字，开始评估版本的不一致性。两种匹配：

  • 严厉匹配：两个版本号完全一致
  • 松弛匹配：两者相互包含就算匹配

创新点

  • 增加字典匹配
  • RE的抽取方法
  • 研究不一致性

重要细节

实验结果和结论

数据集

得到20年的13种类型的78296个CVE ID列表。然后用ID对5个外部网站得到70569个漏洞报告。然后用ID得到78296个NVD项。

手工标注1974个报告，包含5193个ID。

评估

NER
  • 精确率、召回率
  • 消融实验，对于字典匹配
RE
  • 手工标注作为输入的精确率、召回率和准确率
  • NER作为输入的。。。
  • 消融实验，对于字典匹配
对比
  • SemFuzz，使用手工正则表达式抽取版本
  • 端到端的NLP工具
参数实验
迁移学习
  • 对其它12种少量标注类型的数据，微调最后一层

评估信息不一致性

有45.9%严厉匹配，87.3%松弛匹配，NVD和其它来源存在不一致性。

融合方法
  • 取最大值：哪个来源的版本出现次数最多取哪个来源（关注来源）
  • 合并：哪些版本出现次数最多取哪些（关注版本）
