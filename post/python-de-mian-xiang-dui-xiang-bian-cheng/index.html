python的面向对象编程

Posted on 2022-09-05，5 min read

  • 类的实例化
  • 引用类属性

  • 参考资料

对python的类和实例化等做一下梳理。

类的实例化

定义类Human：

class Human(object):
    century = 21

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def speak(self, language):
        print('%s has speak %s ability' % (self.name, language))

    def write(self, word):
        print('%s has write %s ability' % (self.name, word))

    def walk(self):
        print('%s has walk ability' % self)

创建一个实例Johnny：

Johnny = Human('Johnny', 16)

只有将类实例化后，类内部定义的函数才被绑定到实例上，成为实例的方法，否则仅仅只是类内部定义的一个函数，在类中调用的叫做函数，实例化中调用的才称为方法：

print(Human.speak, Human.write, Human.walk)
print(Johnny.speak, Johnny.write, Johnny.walk)

当用实例调用方法时，python解释器会自动把实例对象传入方法中，因此self变量代表了实例对象本身。例如用实例对象Johnny调用三个方法：

Johnny.speak("Chinese")
Johnny.write("Chinese")
Johnny.walk()

假如没有创建实例，直接使用类名去调用speak()等未绑定方法，就需要手动传入self参数，此时self参数并不具备self.name、self.age属性，所以只有walk能调用成功：

Human.walk('Johnny')

引用类属性

__dict__能以字典形式返回对象的属性，键为属性名，值为属性值。类的__dict__存储的是所有实例共享的变量和函数，不包含其父类的属性。实例的__dict__仅存储与该实例相关的实例属性。


century为类属性，可以通过类和实例分别访问：

print(Human.century)
print(Johnny.century)

只有通过类引用时，类属性才能更新它的值，并且类属性的修改会影响到所有的实例；而被实例引用时，会像静态成员那样无法改变其值，因此类变量大多用于存储常量值。

Human.century += 1 # 通过类更新
print(Human.century) # 值改变
print(Johnny.century) # 值改变

Johnny.century += 1 # 通过实例更新
print(Johnny.century) # 通过实例访问，值被改变
print(Human.century) # 通过类访问，值未改变

实际上，上面的代码中，通过实例改变century的值是创建了一个名为century的新实例属性，只是该实例属性刚好与类属性同名。这点和python变量一样，如果变量不存在而对其赋值，就会创建一个变量，并且对其赋值。使用__dict__再次查看实例属性，发现增加了century属性：

其实类和实例都是各自的命名空间，类是类属性的命名空间，实例是实例属性的命名空间，即使名字相同，也是在不同的属性集中，当访问实例属性时，如果不存在该属性，解释器才会从类属性中找。

此处century为不可变变量，如果改为可变变量，如字典类型，就可以通过实例引用century属性且改变其值，不会创建一个同名的实例属性，类属性中的值也会改变。

class Human_A(object):
    century = {"A": 21}
    def __init__(self, name, age):
        self.name = name
        self.age = age


    def speak(self, language):
        print('%s has speak %s ability' % (self.name, language))

    def write(self, word):
        print('%s has write %s ability' % (self.name, word))

    def walk(self):
        print('%s has walk ability' % self)

John = Human_A('John', 22)

print(Human_A.century)
print(John.century)

John.century['B'] = "32" # 通过实例更新

print(Human_A.century)
print(John.century)

引用类的方法

假如要调用类中的方法，但是并不希望这些方法为实例所用，可以使用@staticmethod 或@classmethod语法。在不需要实例化的前提下，可以直接以类名.方法名（）方式来调用。
@staticmethod和@classmethod的区别在于，@staticmethod不需要表示自身对象的self和自身类的cls参数，就和使用函数一样；而@classmethod虽然不需要self参数，但第一个参数需要表示自身类的cls参数。因此在@staticmethod中要调用到这个类的一些属性方法，只能用直接类名.属性名或类名.方法名，而@classmethod因为持有cls参数，可以来调用类的属性、类的方法、实例化对象等。

class Human_B(object):
    century = 21
    name = "John"

    def __init__(self, name, age):
        self.name = name
        self.age = age

    @staticmethod
    def speak(language):
        print('%s has speak %s ability' % (Human_B.name, language))

    def write(self, word):
        print('%s has write %s ability' % (self.name, word))

    @classmethod
    def walk(cls):
        print('%s has walk ability' % cls.name)

Human_B.speak("Chinese")
Human_B.walk()
