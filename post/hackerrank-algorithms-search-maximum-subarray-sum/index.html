HackerRank Algorithms Search: Maximum Subarray Sum

Posted on 2023-04-16，3 min read

题目描述

给你一个大小为N的数组和另外一个整数M。你的目标是找到每个子数组的和对M取余数的最大值。

子数组是指原数组的任意连续元素的子集。

注意你需要找到max(子数组的和）%M ， 其中一共有 N∗(N+1)/2N*(N+1)/2N∗(N+1)/2 个可能的子数组。

思路

  • 暴力解法：用两个循环，依次比较加上后面元素后的大小，维护一个maxnum。时间复杂度 o(n2)o(n^2)o(n2)。
  • 优化：使用Kadane算法。Kadane算法是一种用于解决最大子数组问题的迭代动态规划算法。该问题是在给定数组中寻找具有最大和的连续子数组。Kadane算法的时间复杂度为O(n)，其中n是数组的长度。该算法通过在遍历数组时动态计算最大子数组的和来实现。

伪代码：

max_so_far = 0
max_ending_here = 0

for i in range(len(arr)):
    max_ending_here = max_ending_here + arr[i]
    if max_ending_here < 0:
        max_ending_here = 0
    elif max_so_far < max_ending_here:
        max_so_far = max_ending_here

return max_so_far

max_ending_here是加上当前值后的最大和，max_so_far是总的最大和。遍历数组并将当前元素的值添加到 currSum 并检查, 如果 currSum 大于 maxSum，更新 maxSum 等于 currSum。如果 currSum 小于零，则使 currSum 等于零。

但是题目要求模m，所以普通的这种方法行不通。
我们定义sum[i][j] = (pre[i] - pre[j]) % m , 其中pre[i]是前i个元素相加 % m。j < i，目的是最大化sum[i][j]。
给定i后pre[i]是常数，只考虑pre[j]的情况，有两种情况。

  • pre[j] <= pre[i]，此时目的是找出最小pre[j]。
  • pre[j] > pre[i]，此时目的变为找出最接近pre[i]的pre[j]。因为此时 sum[i][j]为 m+pre[i]-pre[j]，加m是因为pre[i] - pre[j]是负数。
